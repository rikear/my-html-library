@keyframes fillBg {

0% {

background-color: yellow;

}

100% {

background-color: red;

}

}

However, once the animation completes, the background of the div will return to nothing. That's because, by default, when an animation ends, the element goes back to exactly how it was before the animation ran. In order to override this behavior, we have the animation-fill-mode property. In this instance, we could apply this:

animation-fill-mode: forwards;

This makes the item retain any values that have been applied by the animation end. In our case, the div would retain the red background color that the animation ended on. More on the animation-fill-mode property here: https://www.w3.org/TR/css-animations-1/#animation-fill-mode.
