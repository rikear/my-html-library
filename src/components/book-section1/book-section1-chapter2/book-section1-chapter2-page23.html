For example, if an onscreen widget displays a constantly updating stock price, how would a blind user accessing the page know that? WAI-ARIA attempts to solve this problem.

How to implement ARIA fully in something like a web application is outside the scope of this book. However, if that's the kind of project you are building, head over to https:// www.w3.org/ WAI/standards-guidelines/aria/ for more information.

Instead, for a great overview of getting to grips with ARIA, consider reading the W3C Using Aria guide: https://www.w3.org/TR/using-aria/.

The easiest thing you can do to aid assistive technologies is to use the correct elements where possible. A header element is going to be far more useful than < div class-="Header"â€º Similarly, if you have a button on your page, use the button element, rather than a span or other element styled to look like a button.
