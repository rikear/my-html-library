The great news is that all these user interface elements, including the aforementioned slider, placeholder text, spinner, and the input validation, are being handled natively by the browser via HTML5, and no JavaScript.

Let's begin by getting a handle on all the capabilities of HTMLS that relate to forms and make all this possible. Once we understand all the mechanics, we can get to work styling it up.

Understanding the component parts of HTML5 forms

There's a lot going on in our HTML5-powered form, so let's break it down. The three sections of the form are each wrapped in a fieldset, which semantically groups the related sets of form felds, and a legend, which provides the textual explanation of what that fieldset is;

<fieldset>

<legend> About the offending film (part 1 of 3) </legend>

<div>

<label for="film"> The film in question? </label>

<input

id="film"

name="film"

type="text"

placeholder="e.g. King Kong"

required aria-required="true"

/>

</ div>

</fieldset>
